Web terms & anatomy

  • html = hypertext markup language. the language in which webpages/documents are written
    • markup = what the page looks like
  • url = location of a document
  • http = protocol (hypertext transfer) by which a browser makes requests to servers
  • www is basically an enormous (billions) collection of html documents
  • the parts of the internet system:

    • A = the browser program on your computer
    • B = the internet
    • C = http requests
    • D = the server, on which the webpage ultimately resides

  • this is not a webpage - there is no server here, just a file on a computer

Elements & tags

  • anchors (<a>) are for making links
    • usually (always?) they have an attribute "href" with the url as its value
    • the text between the tags is what's written in the page
  • <img> is for images
    • the alt value will be shown if the src value (the image file and path) can't be found/loaded
    • the image path can be a file name if it is stored in the same place as the document. otherwise it will probably be a url
  • there are lots of tags for formatting, eg <b>, <em>, <strong>
  • most elements, including the aforementioned formatted elements, must have closing tags (eg </b>) to mark the end of the element. if omitted, the change invoked by the opening tag will continue indefinitely (the computer is only as smart as the code you write)

Arranging text (basic)

  • unless otherwise instructed, the browser will render the text as a continuous line, wrapping only when it gets to the edge of the window
  • line breaks & white space have to be added with markup
  • many elements, including <img> and <a> are likewise inline
  • <span> is used to define an inline section, eg for applying a set of custom formatting attributes
  • there are also block tags, which define sections with height and width - invisible boxes - eg:
    • <p> defines a new paragraph, with white space between it and the previous paragraph
    • <div> defines a new block, without inserting any white space
  • <br> inserts a line break, but does not create a text block

Coding standards for HTML documents

  • you can just render text in a browser, with or without html, but a complete html doc should contain standard information that the browser needs:
    • <!DOCTYPE html> informs the browser that this is an html document (strictly speaking it means html 5; html 4.01 comes in 3 highly verbose flavours)
    • <html> contains all the actual html code on the page, in 2 subsections:
      • <head> contains metadata, including the page title and references to CSS and script files
      • <body> contains all the content
  • if you create an empty html file Atom has a helpful template that you can use to set up the doc outline
  • html also has indenting standards as described in depth here
